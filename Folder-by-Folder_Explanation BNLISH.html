Project Folder Structure

project_name/
│
├── tests/               
│   └── test_login.py    
│
├── pages/               
│   └── login_page.py    
│
├── utils/               
│   └── browser_utils.py 
│
├── reports/             
│   └── allure_report/   
│
├── screenshots/         
│   └── login_error.png  
│
├── conftest.py          
│
├── requirements.txt     
│
└── pytest.ini

Folder-by-Folder Explanation:

  • tests/ (Test files):
    • Ei folder e sob test files thakbe.
    • Example: test_login.py file jekhane login related tests likha thakbe.
  • pages/ (Page Object files):
    • Ei folder e Page Object Model (POM) er moto files thakbe.
    • Page Object Model mane hocche protyek webpage er ekta class banano, jar modhye oi webpage er actions thakbe.
    • Example: login_page.py file jekhane login page er actions like login form fill kora, submit kora, etc. thakbe.
  • utils/ (Helper functions):
    • Ei folder e utility files thakbe ja test execution e help kore, jemon browser ke handle kora, setup & teardown actions kora, etc.
    • Example: browser_utils.py file jekhane browser initialization, cleanup, or window handling functions thakbe.
  • reports/ (Test result reports):
    • Ei folder e test run er por je reports generate hobe, shegulo store kora hobe.
    • Example: allure_report/ folder thakbe jekhane Allure report related files store hobe.
  • screenshots/ (Captured screenshots during tests):
    • Jodi kono test fail kore, tahole oi time er screenshot capture kora hobe.
    • Ei folder e screenshots save hobe.
    • Example: login_error.png jekhane login error er screenshot thakbe.
  • conftest.py (Shared fixtures for Pytest):
    • Ei file e Pytest er fixtures thakbe, ja test execution er age common setup korbe.
    • Fixtures use kore, amra ekbar setup kore onek test e use korte pari, jemon browser launch kora, database connection setup kora, etc.
  • requirements.txt (Required Python packages):
    • Ei file e amader project e je Python libraries dorkar, segulo list thakbe.
    • Example: selenium, pytest, allure-pytest etc.
  • pytest.ini (Pytest configuration):
    • Ei file e Pytest configuration thakbe, jekhane test er behaviour customize kora jay.
    • Ekhane amra test runs er jonno specific settings provide korte pari, jemon log level, report format, etc.

Required Libraries:

  • selenium==4.10.0 - Selenium WebDriver for browser automation
  • pytest==7.2.2 - Pytest framework for testing
  • allure-pytest==2.10.0 - Allure plugin for Pytest reporting
  • pytest-selenium==1.0.0 - Integration between Selenium and Pytest
  • pytest-xdist==2.5.0 - For parallel test execution
  • pytest-html==3.1.1 - For HTML test report generation
  • webdriver-manager==3.8.3 - For automatic management of WebDriver binaries

  • 2. Virtual environment (venv) setup koro project-er dependencies alada rakhar jonno. Ei command run koro:
    • python -m venv venv
  • 3. Virtual environment activate koro:
    • Windows-er jonno:
      • venv\Scripts\activate
    • macOS/Linux-er jonno:
      • source venv/bin/activate
  • 4. Required Python packages install koro pip install -r requirements.txt diye.
  • 5. Pytest and Selenium use kore test likha suru koro, ar Page Object Model (POM) use koro better organization-er jonno.

  • 7. Chaole Allure ba onno reporting tools integrate kore detailed report generate koro test run-er por.

Conclusion:

Ei structure ta organized r modular.

Test files are separate, page objects keep actions isolated for each page, utilities provide reusable functions, and reports/screenshots keep test results and failure details safe.

pytest.ini and conftest.py help make the tests more structured and easier to maintain across larger projects.

By following this folder structure and best practices, you ensure your project remains clean, scalable, and easy to maintain. The separation of concerns, reuse of page objects, and use of helper functions make it easier to extend the tests as your project grows.

Keep your test suite organized, report test results clearly, and ensure that your automation setup is easily shareable and reproducible across different environments.
